# Setup & Installation Guide

## Prerequisites

- **Flutter SDK** >= 3.3.0 (Dart >= 3.3.0)
- **Android Studio** or **VS Code** with Flutter extensions
- **Firebase CLI** (`npm install -g firebase-tools`)
- **FlutterFire CLI** (`dart pub global activate flutterfire_cli`)
- A **Firebase project** (project ID: `golden-care-d4863` or your own)

### Platform-Specific Requirements

| Platform | Requirements |
|---|---|
| Android | Android SDK, JDK 17+, `key.properties` with signing config |
| iOS | Xcode 15+, CocoaPods, Apple Developer account |
| Web | Chrome or any modern browser |

---

## Installation

### 1. Clone the Repository

```bash
git clone <repository-url>
cd Golden_Care_App
```

### 2. Install Dependencies

```bash
flutter pub get
```

### 3. Firebase Configuration

The project requires Firebase services. Configuration files needed:

- **`lib/firebase_options.dart`** - Already included (auto-generated by FlutterFire CLI)
- **`android/app/google-services.json`** - Required for Android builds
- **`ios/Runner/GoogleService-Info.plist`** - Required for iOS builds

To regenerate for your own Firebase project:

```bash
flutterfire configure --project=<your-firebase-project-id>
```

### 4. Firebase Remote Config

The following Remote Config parameters must be set in the Firebase console:

| Key | Type | Description |
|---|---|---|
| `razorpay_key_id` | string | Razorpay API key for payment processing |
| `maps_api_key` | string | Google Maps API key (required for web geocoding) |
| `vapid_key` | string | VAPID key for Firebase Cloud Messaging on web |
| `booking_start_hour` | number | Start of booking window (24h format, e.g., 8) |
| `booking_end_hour` | number | End of booking window (24h format, e.g., 20) |

### 5. Google Maps API Key (Web)

For web builds, add your Google Maps API key in `web/index.html`:

```html
<script src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY"></script>
```

### 6. Android Signing (Release Builds)

Create `android/key.properties`:

```properties
storePassword=<your-store-password>
keyPassword=<your-key-password>
keyAlias=upload
storeFile=<path-to-upload-keystore.jks>
```

## Deployment

### Web (Firebase Hosting)

```bash
# Build the web app
flutter build web --release

# Deploy to Firebase Hosting
firebase deploy --only hosting
```

### Firestore Rules

```bash
firebase deploy --only firestore:rules
```

### Storage Rules

```bash
firebase deploy --only storage
```

### Remote Config

```bash
firebase deploy --only remoteconfig
```

### Deploy Everything

```bash
flutter build web --release && firebase deploy
```

## Firebase Services Setup

Ensure the following Firebase services are enabled in your project console:

1. **Authentication** - Enable Email/Password, Google, and Phone providers
2. **Cloud Firestore** - Create database in production mode, deploy security rules
3. **Firebase Storage** - Enable and deploy storage rules
4. **Cloud Messaging** - Enable for push notifications
5. **Remote Config** - Set required parameters (see section 4 above)
6. **App Check** - Enable reCAPTCHA Enterprise for web

## Seeding Initial Data

### Services

Services must be added to the `services` Firestore collection manually or via a seed script. Each service document should follow this structure:

```json
{
  "title": "Home Care",
  "description": "Professional home care service",
  "category": "care",
  "imageUrl": "https://example.com/image.jpg",
  "includedItems": ["Vital signs monitoring", "Medication management"],
  "isPopular": true,
  "options": [
    { "duration": "4 hours", "price": 999 },
    { "duration": "8 hours", "price": 1799 }
  ]
}
```

### Whitelisted Partners

To allow caregiver signups, add their email to `whitelisted_partners`:

```json
// Document ID = email address
{
  "isEnabled": true
}
```

### System Config

Create a `config/system` document:

```json
{
  "bookingStartHour": 8,
  "bookingEndHour": 20,
  "profilePhotoRequired": false
}
```

## Troubleshooting

| Issue | Solution |
|---|---|
| `google-services.json` not found | Run `flutterfire configure` or download from Firebase console |
| Google Maps not showing on web | Ensure Maps API key is set in `web/index.html` and Google Maps JavaScript API is enabled |
| Razorpay not working | Verify `razorpay_key_id` is set in Firebase Remote Config |
| FCM not working on web | Set `vapid_key` in Remote Config and ensure `firebase-messaging-sw.js` exists in `web/` |
| Phone auth not working | Enable Phone provider in Firebase Auth console; for web, reCAPTCHA must be configured |
| Build fails on iOS | Run `cd ios && pod install --repo-update` |
| Geocoding fails on web | Ensure `maps_api_key` is configured in Remote Config with Geocoding API enabled |
